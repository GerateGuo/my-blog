Pyserini 信息检索实验：BM25/BGE/ColBERT/SPLADE-v3 四模型对比

概述

最近完成了《大语言模型驱动的智能信息检索技术及应用》课程的期末实验，在 Pyserini 框架下系统对比了四种检索模型——BM25（传统稀疏）、BGE（稠密Bi-encoder）、ColBERT（后期交互）和 SPLADE-v3（学习型稀疏）——在四个标准数据集上的表现。

实验环境

设备 配置
Windows Ryzen 9950X (16C/32T), 32GB RAM, RTX 5070 Ti 12GB
Mac M5 Apple Silicon M5, 24GB 统一内存

关键发现

1. BGE 稠密检索全面最优

BGE 在所有数据集上一致优于 BM25，提升幅度 15%~77%。在 MS MARCO 上通过重排序策略（BM25 top-100 + BGE 重排）实现了 MRR=0.3261，远超 BM25 的 0.184。

2. SPLADE-v3 高召回高速度

SPLADE-v3 在 nfcorpus 上 MAP=0.4152（BM25 的 2.6 倍），索引仅 1.8MB，1 秒跑完 323 查询。但 NDCG@10 偏低，属于典型的”高召回、低精度排序”模型。

3. ColBERT 优化经验

通过 L2 归一化、MPS float16 混合精度和批量预编码，NDCG@10 从 0.55 提升到 0.76（+39%）。

4. 多线程性能

BM25 多线程搜索：16 线程共享 searcher 实现 4.5x 加速（225s → 50s），32 线程收益递减至 5.0x。

5. 领域迁移问题

tct_colbert-v2 在 MS MARCO 上 MAP=0.3509，但迁移到医学领域降为 0.08，说明领域微调至关重要。
